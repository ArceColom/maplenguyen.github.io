RESOLVING CONFLICTS

Conflicts occur when changes clash with one another in the same line, or when one person deletes a file while another try to alter it.

These are the steps to fix such occurence:


1. Open the terminal.
2. Direct into the Git repository that has the merge conflict:

cd _REPOSITORY NAME_

3.Decide if you want to keep the removed file. You can view the latest changes made to the removed file.
To add the removed file back to repository:

$ git add README.md

To remove this file from repository:

$ git rm README.md
> README.md: needs merge
> rm 'README.md'

Finally, don't forget to commit your changes with a comment to let your group know.
